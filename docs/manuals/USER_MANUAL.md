## はじめに

### システム概要

AIインタビューシステムは、内部監査、リスク管理、コンプライアンス業務におけるインタビュー業務を自動化・効率化するプラットフォームです。AI技術を活用し、インタビューの実施から報告書作成まで一貫してサポートします。

**主な機能**
- 案件・タスクの一元管理
- 質問テンプレートの作成・管理
- AIによるリアルタイムインタビュー
- 音声・テキスト両対応の入力機能
- 自動レポート生成とワークフロー管理
- 過去データのセマンティック検索

**対象業務領域（GRC）**
- **Governance（ガバナンス）**: 取締役会評価、暗黙知ヒアリング
- **Risk（リスク管理）**: リスクアセスメント、プロセスレビュー
- **Compliance（コンプライアンス）**: コンプライアンス調査、内部通報調査、統制評価

### 動作環境

**推奨ブラウザ**
- Google Chrome（最新版）
- Microsoft Edge（最新版）
- Mozilla Firefox（最新版）
- Safari（最新版）

**推奨環境**
- 画面解像度: 1920×1080以上
- インターネット接続: 安定した高速回線
- 音声入力使用時: マイク・スピーカー

### ログイン方法

1. ブラウザでシステムURLにアクセス
2. メールアドレスとパスワードを入力
3. 多要素認証（MFA）が有効な場合、認証コードを入力
4. 「ログイン」ボタンをクリック

**初回ログイン時**
- システム管理者から送付された初期パスワードを使用
- ログイン後、パスワード変更を推奨

## ダッシュボード

### 画面構成

ダッシュボードはシステムのホーム画面で、業務の全体像を把握できます。

**主要エリア**
- **統計情報パネル**: 重要指標を4つのカードで表示
- **最近の活動**: 直近のタスクやインタビュー履歴
- **クイックアクション**: よく使う機能へのショートカット
- **通知センター**: システムからの重要なお知らせ

### 統計情報の見方

ダッシュボード上部に表示される4つの主要指標:

| 指標 | 説明 | 活用方法 |
|------|------|----------|
| **アクティブ案件数** | 現在進行中の案件数 | 業務負荷の把握 |
| **タスク数** | 未完了タスクの総数 | 優先度の判断 |
| **完了インタビュー数** | 実施済みインタビュー数 | 進捗状況の確認 |
| **生成レポート数** | 作成済みレポート数 | 成果物の管理 |

**トレンド表示**
- 各指標の増減を矢印とパーセンテージで表示
- 緑色上向き矢印: 増加傾向
- 赤色下向き矢印: 減少傾向

### クイックアクション

よく使う操作を素早く実行できます:

- **新規案件作成**: 案件登録画面を開く
- **インタビュー開始**: 実行可能なタスク一覧へ
- **レポート閲覧**: 最新レポート一覧へ
- **検索**: ナレッジベース検索画面へ

## 案件管理

### 案件の作成

新規案件を登録する手順:

1. サイドバーから「案件」メニューをクリック
2. 右上の「新規案件作成」ボタンをクリック
3. 必要情報を入力:
   - **案件名**: 識別しやすい名称（例: 2026年度リスクアセスメント）
   - **クライアント名**: 対象部署・会社名
   - **クライアント情報**: 連絡先、担当者情報
   - **説明**: 案件の目的・背景（任意）
   - **開始日**: 案件開始予定日
   - **終了日**: 案件完了予定日（任意）
4. 「作成」ボタンをクリック

### 案件の編集

既存案件の情報を変更する:

1. 案件一覧から対象案件をクリック
2. 案件詳細画面右上の「編集」ボタンをクリック
3. 変更したい項目を修正
4. 「保存」ボタンをクリック

**編集可能項目**
- 案件名、クライアント名、説明
- 開始日、終了日
- ステータス（後述）

### 案件ステータス管理

案件は3つのステータスで管理されます:

| ステータス | 意味 | 用途 |
|-----------|------|------|
| **Active（進行中）** | 現在作業中の案件 | 日常業務の管理 |
| **Completed（完了）** | 業務が終了した案件 | 完了案件の記録 |
| **Archived（アーカイブ）** | 保管・参照用 | 過去案件の整理 |

**ステータス変更方法**
1. 案件詳細画面で「ステータス変更」をクリック
2. 新しいステータスを選択
3. 変更理由を入力（任意）
4. 「確定」ボタンをクリック

### 案件の検索・フィルタリング

大量の案件から目的のものを素早く見つける:

**検索機能**
- 検索バーに案件名、クライアント名、説明文を入力
- リアルタイムで結果が絞り込まれる

**フィルタ機能**
- ステータスフィルタ: Active/Completed/Archivedで絞り込み
- 期間フィルタ: 開始日・終了日で絞り込み
- ソート: 作成日、更新日、案件名で並び替え

## タスク管理

### タスクの作成

案件に紐づくタスクを作成:

1. 案件詳細画面から「タスク」タブを選択
2. 「新規タスク作成」ボタンをクリック
3. タスク情報を入力:
   - **タスク名**: 具体的な業務名
   - **ユースケースタイプ**: 業務の種類を選択（下記参照）
   - **説明**: 実施内容の詳細
   - **担当者**: タスク実施者
   - **期限**: 完了予定日
4. 「作成」ボタンをクリック

### ユースケースタイプ

タスクは7つの業務タイプに分類されます:

| ユースケース | 説明 | 使用例 |
|------------|------|--------|
| **compliance_survey** | コンプライアンス調査 | 法令遵守状況の確認、違反事例の調査 |
| **whistleblower_investigation** | 内部通報調査 | 通報内容の事実確認、関係者ヒアリング |
| **process_review** | プロセスレビュー | 業務プロセスの効率性・有効性評価 |
| **control_evaluation** | 統制評価 | 内部統制の整備・運用状況評価 |
| **risk_assessment** | リスクアセスメント | リスクの識別・分析・評価 |
| **board_effectiveness** | 取締役会評価 | 取締役会の実効性評価インタビュー |
| **tacit_knowledge** | 暗黙知ヒアリング | ベテラン社員の知見・ノウハウの抽出 |

### タスクステータス遷移

タスクは以下のライフサイクルで管理されます:

```
Pending（未着手） → In Progress（進行中） → Completed（完了）
```

**ステータス変更タイミング**
- **Pending → In Progress**: インタビュー開始時に自動遷移
- **In Progress → Completed**: インタビュー終了・レポート承認時

**手動変更**
1. タスク詳細画面で現在のステータスをクリック
2. 変更先ステータスを選択
3. 確定

### タスクの優先度管理

効率的な業務遂行のための優先度設定:

- **高**: 期限が近い、重要度が高い
- **中**: 通常の業務タスク
- **低**: 期限に余裕がある、補助的業務

**フィルタリング**
- 「自分のタスク」: 自分が担当のタスクのみ表示
- 「期限順」: 期限が近い順にソート
- 「ステータス別」: 未着手・進行中で絞り込み

## テンプレート管理

### テンプレート概要

質問テンプレートは、インタビューで使用する質問セットを事前に定義します。ユースケースタイプごとにテンプレートを作成・管理できます。

### 新規テンプレート作成

1. サイドバーから「テンプレート」メニューをクリック
2. 「新規テンプレート作成」ボタンをクリック
3. 基本情報を入力:
   - **テンプレート名**: 識別しやすい名称
   - **ユースケースタイプ**: 適用業務を選択
   - **説明**: テンプレートの用途・特徴
4. 「作成」ボタンをクリック

### 質問の追加・編集

テンプレートに質問を追加:

1. テンプレート詳細画面で「質問を追加」をクリック
2. 質問内容を入力:
   - **質問文**: インタビューで尋ねる内容
   - **質問タイプ**: 選択式、記述式、数値など
   - **必須/任意**: 回答の必須設定
   - **ヒント**: 回答者へのガイダンス
3. 「追加」ボタンをクリック

**編集**
- 既存質問をクリックして内容を修正
- 「保存」で変更を確定

**削除**
- 質問右側のゴミ箱アイコンをクリック
- 確認ダイアログで「削除」を選択

### フォローアップ質問の設定

特定の回答に応じた追加質問を自動表示:

1. 親質問の「フォローアップ追加」をクリック
2. トリガー条件を設定:
   - **条件タイプ**: 特定の回答、キーワード含む、など
   - **条件値**: 具体的な値や文言
3. フォローアップ質問を入力
4. 「保存」ボタンをクリック

**例**
- 親質問: 「内部統制に不備はありますか？」
- 条件: 回答が「はい」
- フォローアップ: 「具体的にどのような不備ですか？」

### 質問の並び替え

ドラッグ&ドロップで質問順序を変更:

1. テンプレート編集画面で「並び替えモード」を有効化
2. 質問カードをドラッグして目的の位置にドロップ
3. 「順序を保存」ボタンをクリック

**ベストプラクティス**
- 基本情報から詳細へ段階的に質問
- 関連する質問をグルーピング
- 重要度の高い質問を前半に配置

### テンプレートの公開

作成したテンプレートを使用可能にする:

**ステータス**
- **Draft（下書き）**: 編集中、インタビューには使用不可
- **Published（公開）**: インタビューで使用可能

**公開手順**
1. テンプレート詳細画面で「公開」ボタンをクリック
2. 確認ダイアログで内容を確認
3. 「公開する」を選択

**注意事項**
- 公開後も編集可能
- 公開中のテンプレートを変更すると、新規インタビューから適用
- 実施中のインタビューには影響しない

## インタビュー実施

### インタビューの開始

1. ダッシュボードまたはタスク一覧から対象タスクを選択
2. 「インタビュー開始」ボタンをクリック
3. インタビュー設定を確認:
   - 使用テンプレート
   - インタビュイー情報
   - 記録方法（テキスト/音声）
4. 「開始」ボタンをクリック

**自動処理**
- タスクステータスが「In Progress」に遷移
- WebSocket接続でリアルタイム通信開始
- AIエージェントが初期化

### テキスト入力モード

キーボードで質問と回答を入力する標準モード:

**使い方**
1. AIからの質問が画面に表示される
2. 回答入力欄に回答を入力
3. Enterキーまたは「送信」ボタンで送信
4. AIが回答を分析し、次の質問を生成

**機能**
- **リアルタイム入力補完**: 過去の回答を参考に候補表示
- **下書き保存**: 入力中の内容を自動保存
- **履歴表示**: 過去の質問・回答をスクロールで確認

### 音声入力モード

マイクを使った音声での対話形式:

**初回設定**
1. インタビュー画面で「音声モード」を選択
2. マイクへのアクセス許可を承認
3. 音声テストで動作確認

**使い方**
1. AIの質問が音声で再生される
2. マイクボタンを押しながら回答を話す
3. ボタンを離すと音声がテキスト化され送信
4. AIが分析し、次の質問へ

**音声認識のコツ**
- 静かな環境で実施
- マイクから20-30cm程度の距離
- はっきりとした発音で話す
- 専門用語は事前に辞書登録

### インタビューの一時停止

途中で中断し、後で再開する:

1. 画面右上の「一時停止」ボタンをクリック
2. 確認ダイアログで「一時停止」を選択
3. それまでの内容が自動保存される

**再開方法**
1. タスク一覧から対象タスクを選択
2. 「インタビュー再開」ボタンをクリック
3. 中断時点から継続

**保存される情報**
- 質問・回答の履歴
- 現在の質問番号
- AIの分析結果
- メモ・注釈

### インタビューの終了

インタビューを完了する:

1. すべての質問が終了、または必要十分な情報を収集
2. 「インタビュー終了」ボタンをクリック
3. 終了確認画面で内容をレビュー
4. 「終了して保存」を選択

**自動処理**
- 全データが確定保存
- AIが要約レポートを生成
- タスクステータスが「Completed」に遷移
- ナレッジベースにデータが蓄積

### リアルタイムAI対話の仕組み

**WebSocketベース通信**
- サーバーとブラウザ間で双方向リアルタイム通信
- 低遅延でスムーズな対話を実現

**AIの動作**
1. ユーザー回答を受信
2. 回答内容を分析・理解
3. テンプレートと文脈を考慮
4. 最適な次の質問を生成
5. 必要に応じてフォローアップ質問を追加

**動的質問生成**
- 回答内容に応じて質問をカスタマイズ
- 不明瞭な回答には掘り下げ質問
- 十分な情報が得られたら次のトピックへ

## レポート機能

### レポートタイプ

インタビュー結果から6種類のレポートを生成できます:

| タイプ | 説明 | 主な用途 |
|--------|------|----------|
| **Summary（要約）** | インタビュー内容の要点まとめ | 経営層への報告、概要把握 |
| **Process Document（プロセス文書）** | 業務プロセスの詳細記述 | 業務マニュアル、手順書作成 |
| **RCM（リスク統制マトリクス）** | リスクと統制の対応表 | 内部統制文書、監査調書 |
| **Audit Workpaper（監査調書）** | 監査手続きと証拠の記録 | 監査ファイル、証跡管理 |
| **Compliance Report（コンプライアンス報告書）** | 法令遵守状況の報告 | 規制当局報告、社内報告 |
| **Analysis（分析レポート）** | データ分析と考察 | 改善提案、意思決定支援 |

### レポートの生成

1. 完了したインタビューの詳細画面を開く
2. 「レポート生成」ボタンをクリック
3. レポートタイプを選択
4. 生成オプションを設定:
   - **言語**: 日本語/英語/中国語
   - **詳細度**: 簡易/標準/詳細
   - **含める要素**: 図表、統計、推奨事項など
5. 「生成開始」ボタンをクリック

**生成時間**
- 通常: 30秒～2分
- 大量データの場合: 最大5分

### レポートワークフロー

レポートは4段階のステータスで管理されます:

```
Draft（下書き） → Review（レビュー中） → Approved（承認済） → Published（公開）
```

**各ステータスの役割**

1. **Draft（下書き）**
   - AI生成直後の状態
   - 担当者が内容を確認・編集
   - 他のメンバーには非公開

2. **Review（レビュー中）**
   - レビュアーに内容確認を依頼
   - コメント・修正指示が可能
   - 承認/差し戻しの判断待ち

3. **Approved（承認済）**
   - レビュアーが内容を承認
   - 公開前の最終確認済み状態
   - 軽微な修正のみ可能

4. **Published（公開）**
   - 正式な成果物として確定
   - 関係者への共有が可能
   - 編集不可（新版作成で対応）

### レポートの編集

生成されたレポートを修正・改善:

1. レポート詳細画面で「編集」ボタンをクリック
2. リッチテキストエディタで内容を編集:
   - 文章の修正・追加
   - 見出し・箇条書きの整形
   - 表・図の挿入・編集
3. 「保存」ボタンで変更を保存

**編集可能範囲**
- Draft、Review: 自由に編集可
- Approved: 軽微な修正のみ（誤字脱字など）
- Published: 編集不可

### レポートのレビュー

品質担保のためのレビュープロセス:

**レビュー依頼**
1. Draft状態のレポートで「レビュー依頼」をクリック
2. レビュアーを選択
3. 依頼コメントを入力
4. 「送信」ボタンをクリック

**レビュー実施**
1. レビュー依頼通知を受け取る
2. レポート内容を確認
3. コメントを付与:
   - 特定箇所を選択してコメント挿入
   - 全体コメントを記入
4. 判断を選択:
   - **承認**: 問題なし、承認済に遷移
   - **修正依頼**: 指摘事項あり、Draftに差し戻し

### レポートのエクスポート

レポートを外部ファイルとしてダウンロード:

**対応形式**
- **PDF**: 印刷・配布用の確定版
- **Word（.docx）**: 編集可能な文書形式
- **Excel（.xlsx）**: 表形式データ、RCMなど

**エクスポート手順**
1. レポート詳細画面で「エクスポート」をクリック
2. 出力形式を選択
3. オプション設定:
   - ページ設定（PDF/Word）
   - シート構成（Excel）
   - ヘッダー・フッターの有無
4. 「ダウンロード」ボタンをクリック

**ファイル名規則**
```
[レポートタイプ]_[案件名]_[生成日].[拡張子]
例: Summary_2026年度リスクアセスメント_20260215.pdf
```

## ナレッジ検索

### セマンティック検索とは

キーワード完全一致ではなく、意味的な類似性に基づいて過去のインタビューデータを検索する機能です。

**従来の検索との違い**
- **キーワード検索**: 「内部統制」という単語を含む文書のみヒット
- **セマンティック検索**: 「内部統制」「統制活動」「コントロール」など意味的に関連する内容もヒット

### 検索の実行

1. サイドバーから「ナレッジ検索」メニューをクリック
2. 検索バーに調べたい内容を入力:
   - 単語だけでなく、文章でも検索可能
   - 例: 「売掛金の回収プロセスにおける承認手続き」
3. Enterキーまたは検索ボタンをクリック
4. 検索結果が関連度順に表示される

**検索のコツ**
- 具体的な業務内容や課題を文章で記述
- 関連キーワードを複数含める
- 質問形式でも検索可能

### 検索結果の見方

検索結果は以下の情報とともに表示されます:

**結果カード構成**
- **タイトル**: インタビュー名・案件名
- **関連度スコア**: 検索クエリとの類似度（0-100%）
- **抜粋**: 関連する質問・回答のハイライト
- **メタデータ**: 実施日、担当者、ユースケースタイプ

**ソート・フィルタ**
- 関連度順/日付順で並び替え
- ユースケースタイプで絞り込み
- 期間指定で検索範囲を限定

### 検索結果の活用

見つけた情報を業務に活用:

1. 結果カードをクリックして詳細を表示
2. 関連する質問・回答を全文閲覧
3. 有用な情報をメモやレポートにコピー
4. 「類似インタビュー」から関連案件を探索

**ナレッジの蓄積**
- 実施したインタビューは自動的にナレッジベースに追加
- 検索精度は蓄積データ量に応じて向上
- 組織の知見が体系的に蓄積

## 設定

### プロフィール管理

個人情報とアカウント設定:

1. 画面右上のユーザーアイコンをクリック
2. 「設定」メニューを選択
3. 「プロフィール」タブで編集:
   - **表示名**: システム上の表示名
   - **メールアドレス**: ログインID（変更時は確認メール送信）
   - **電話番号**: 連絡先（任意）
   - **所属部署**: 組織・チーム名
   - **役職**: 監査役、リスクマネージャーなど
4. 「保存」ボタンをクリック

**パスワード変更**
1. プロフィール画面で「パスワード変更」をクリック
2. 現在のパスワードを入力
3. 新しいパスワードを入力（2回）
4. 「変更」ボタンをクリック

**パスワード要件**
- 最低8文字以上
- 大文字・小文字・数字・記号を各1文字以上
- 過去3回のパスワードは使用不可

### AIプロバイダー設定

使用するAIサービスの選択と設定:

**対応プロバイダー**
- **Azure OpenAI**: Microsoft Azure経由でOpenAIモデルを使用
- **AWS Bedrock**: AmazonのマネージドAIサービス
- **GCP Vertex AI**: Google CloudのAIプラットフォーム

**設定手順**
1. 「設定」メニューから「AIプロバイダー」タブを選択
2. 使用するプロバイダーを選択
3. 認証情報を入力:
   - **Azure OpenAI**: エンドポイント、APIキー、デプロイメント名
   - **AWS Bedrock**: アクセスキーID、シークレットキー、リージョン
   - **GCP Vertex AI**: プロジェクトID、サービスアカウントキー
4. 「接続テスト」で動作確認
5. 「保存」ボタンをクリック

**モデル選択**
- GPT-4、Claude、Geminiなど利用可能モデルを選択
- 用途に応じて最適なモデルを推奨表示

**注意事項**
- APIキーなどの認証情報は暗号化して保存
- システム管理者が組織レベルで一括設定も可能

## FAQ・用語集

### よくある質問（FAQ）

**Q1: インタビュー中に接続が切れた場合はどうなりますか？**

A: WebSocket接続が切れた場合、システムは自動的に再接続を試みます。再接続に失敗した場合でも、それまでの内容は自動保存されているため、再ログイン後に「インタビュー再開」から継続できます。

**Q2: 複数のタスクで同じテンプレートを使用できますか？**

A: はい、可能です。公開済みテンプレートは何度でも再利用できます。テンプレートを編集しても、既に開始されているインタビューには影響しません。

**Q3: レポートを公開後に修正したい場合はどうすればよいですか？**

A: 公開済みレポートは直接編集できません。新しいバージョンとして作成し直す必要があります。レポート詳細画面で「新版作成」を選択し、編集後に再度承認プロセスを経て公開してください。

**Q4: 音声認識の精度が低い場合の対処法は？**

A: 以下を確認してください:
- マイクの接続と設定（入力レベル）
- 周囲の騒音レベル
- マイクとの距離（20-30cm推奨）
- ブラウザのマイク許可設定
- 専門用語は事前に「用語辞書」に登録

**Q5: ナレッジ検索で結果が見つからない理由は？**

A: 以下の原因が考えられます:
- システムに蓄積されているデータがまだ少ない
- 検索クエリが抽象的すぎる/具体的すぎる
- 該当するユースケースタイプのデータがない
より具体的な文章で検索するか、フィルタ条件を緩めてみてください。

**Q6: 複数人で同時に同じ案件のタスクを実施できますか？**

A: 可能です。案件内の異なるタスクであれば、複数のメンバーが同時に作業できます。ただし、同一タスクの同時編集はできません。

**Q7: レポートのエクスポートで図表が正しく表示されない場合は？**

A: PDF/Word出力時に複雑な図表が崩れる場合があります。その場合は:
1. ブラウザの印刷機能を使用（Ctrl+P）
2. 別の出力形式（PDFとWordを切り替え）を試す
3. 図表を個別に画像として保存し、手動で挿入

**Q8: MFA（多要素認証）を設定/解除するには？**

A: プロフィール設定から「セキュリティ」タブを選択し、「MFA設定」から有効化/無効化できます。初回設定時は認証アプリ（Google Authenticator、Microsoft Authenticatorなど）が必要です。

**Q9: 削除した案件やレポートは復元できますか？**

A: はい、削除後30日間は「ごみ箱」に保管されます。サイドバーの「ごみ箱」から対象を選択し、「復元」ボタンで元に戻せます。30日経過後は完全削除され、復元できません。

**Q10: システムのメンテナンス予定を確認するには？**

A: ダッシュボードの通知センター、または「設定」→「システム情報」でメンテナンススケジュールを確認できます。メンテナンス前には事前にメール通知も送信されます。

### 用語集

**案件（Project）**
監査、調査、評価などの業務単位。複数のタスクをグループ化して管理する最上位の概念。

**タスク（Task）**
案件内の個別業務。1つのタスクに対して1つのインタビューを実施。

**ユースケース（Use Case）**
業務の種類・目的を分類するカテゴリ。全7種類（コンプライアンス調査、内部通報調査など）。

**テンプレート（Template）**
インタビューで使用する質問セットの雛形。ユースケースごとに作成・管理。

**フォローアップ質問（Follow-up Question）**
回答内容に応じて動的に表示される追加質問。深掘りや明確化に使用。

**インタビュー（Interview）**
テンプレートに基づいてAIと対話しながらデータを収集するプロセス。

**WebSocket**
サーバーとクライアント間の双方向リアルタイム通信を実現する技術。インタビューの対話に使用。

**セマンティック検索（Semantic Search）**
意味的な類似性に基づく検索。キーワード完全一致ではなく、文脈を理解して関連情報を検索。

**ナレッジベース（Knowledge Base）**
過去のインタビューデータが蓄積されたデータベース。検索や分析に活用。

**レポートワークフロー（Report Workflow）**
Draft→Review→Approved→Publishedの4段階で品質を管理する仕組み。

**RCM（Risk and Control Matrix）**
リスクと統制を紐付けて管理する表形式の文書。内部統制で使用。

**GRC（Governance, Risk, Compliance）**
ガバナンス、リスク管理、コンプライアンスの総称。本システムの対象領域。

**MFA（Multi-Factor Authentication）**
多要素認証。パスワードに加えて認証アプリなどの第2要素で本人確認。

**AIプロバイダー（AI Provider）**
AIサービスを提供するクラウド事業者。Azure OpenAI、AWS Bedrock、GCP Vertex AIなど。

**ステータス（Status）**
案件・タスク・レポートの現在の状態を示す分類。進行管理に使用。

**アーカイブ（Archive）**
完了した案件を保管状態にすること。一覧には表示されないが検索・参照は可能。
